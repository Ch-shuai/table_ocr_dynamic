# 给 Codex / OpenClaw 的二次开发说明

请在当前项目基础上继续开发。项目目标是识别股票行情截图，将每一行解析为一个个股对象，并按表头字段输出 CSV、Excel、JSONL 和每个股票单独 JSON。

重要约束：

1. 不允许使用固定绝对像素坐标作为核心逻辑。
2. 本项目是“固定结构表格识别”，不是“固定像素模板识别”。
3. 表头字段、字段顺序、字段类型固定；表格区域、列边界、行边界必须运行时动态计算。
4. 优先通过 OpenCV 检测表格横线/竖线。
5. 竖线检测失败时，通过 OCR 表头中心点推断列边界。
6. 横线检测失败时，通过代码列 6 位股票代码的 y 坐标推断行边界。
7. 每个单元格必须单独 OCR。
8. 每一行必须组装为一个 StockRecord。
9. 股票代码必须校验为 6 位数字。
10. 股票名称必须优先通过 A 股代码名称字典修正。
11. 所有低置信度、格式错误、字段为空、代码名称不匹配的单元格必须输出异常报告。

优先开发顺序：

1. 运行 `--detect-only`，优化动态表格线检测。
2. 运行 `--debug-cells`，检查单元格裁剪是否准确。
3. 优化 PaddleOCR 对小数字、百分号、负号、万/亿单位的识别。
4. 增加完整 A 股代码名称表。
5. 增加 Web 复核页面。
6. 增加行情源二次校验。
